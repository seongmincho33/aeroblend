# 마우스 에임 시각적 가이드

## 개요

War Thunder 스타일 마우스 에임 조종의 시각적 피드백 구조를 설명합니다.

## HUD 요소 3가지

| 요소 | 모양 | 의미 |
|------|------|------|
| 하얀 동그라미 | ◯ | 마우스 에임 목표점 — "여기로 가고 싶다" |
| 초록 십자선 | ✚ | 기체 노즈 방향 — "지금 여기 보고 있다" |
| 하얀 가이드선 | │ | 목표점과 노즈를 잇는 선 — "당기는 줄" |

## 조종 원리: "당기기"

사용자는 하얀 가이드선으로 초록 십자선을 당깁니다.

```
  마우스 이동 방향
       ↓
    ◯ ← 하얀 동그라미 (목표점)
    |
    | ← 하얀 가이드선 ("당기는 줄")
    |
    ✚ ← 초록 십자선 (기체 노즈)
```

1. 마우스를 움직이면 → 하얀 동그라미가 먼저 이동
2. 가이드선이 늘어남 → 기체가 PID 제어로 따라감
3. 초록 십자선이 하얀 동그라미 쪽으로 당겨짐
4. 결국 초록 십자선이 하얀 동그라미 안에 들어옴 (수렴)

낚싯줄처럼 목표점을 던지고, 기체가 끌려오는 느낌.

## 카메라 동작: 반대 방향 회전

카메라는 항공기와 십자선이 움직이는 **반대 방향**으로 이동합니다.
그 결과, 하얀 동그라미(마우스 에임)는 항상 화면 중앙에 위치합니다.

```
[화면]
┌─────────────────────────────┐
│                             │
│         ◯ ← 하얀 동그라미    │  ← 항상 화면 중앙
│         |   (마우스 에임)     │
│         |                   │
│     ✚ ←── 초록 십자선        │  ← 아직 따라가는 중
│   ✈ (항공기)                 │
│                             │
└─────────────────────────────┘
       ↑ 카메라는 항공기+십자선이
         움직이는 반대로 회전
```

### 왜 이렇게 되는가

카메라 위치 = `aircraft_pos - aim_dir × distance`

카메라는 에임 방향의 뒤쪽에서 항공기를 바라봅니다:

- **하얀 동그라미** (에임 목표) = 카메라 정면 방향 = 항상 화면 중앙
- **항공기 + 초록 십자선** = 아직 에임 방향에 도달하지 못했으면 화면 중앙에서 벗어나 보임
- 기체가 따라가면서 → 초록 십자선이 중앙으로 수렴 → 하얀 동그라미와 겹침

### 핵심 원칙

**"마우스 = 화면 중앙 = 내가 보고 싶은 곳"**

이 등식이 항상 성립하고, 항공기가 거기로 끌려옵니다.

## 조종 흐름 요약

```
[사용자 마우스 이동]
       ↓
[하얀 동그라미 이동] → 화면 중앙에 유지 (카메라 반대 회전)
       ↓
[가이드선 늘어남]
       ↓
[PID 제어기 작동] → 기체가 목표 방향으로 선회
       ↓
[초록 십자선 이동] → 하얀 동그라미에 수렴
       ↓
[가이드선 줄어듦] → 안정 상태
```
